# Import Wizard UX Flow

**Feature:** FC-026 - Data Import System (UI/UX Design)  
**Author:** Architect  
**Date:** 2025-01-26  
**Status:** Design Phase

---

## Overview

This document describes the user experience flow for the Import Wizard, which guides users through importing financial data from YNAB or spreadsheet files.

### Design Principles
- **Progressive Disclosure:** Show complexity only when needed
- **Clear Feedback:** Always tell the user what's happening and why
- **Error Recovery:** Make it easy to fix mistakes without starting over
- **Familiarity:** Use patterns from tools users already know (ShareGate, Google Sheets import)

---

## Accessibility Features

### Keyboard Navigation
- Tab through all interactive elements
- Enter/Space to activate buttons
- Escape to close dialogs
- Arrow keys in dropdown menus

### Screen Reader Support
- ARIA labels on all form fields
- ARIA live regions for progress updates
- Descriptive link text (not "Click here")
- Alt text for status icons

### Visual Accessibility
- High contrast mode support
- Color not sole indicator (use icons + color)
- Minimum font size: 14px body text
- Focus indicators on all interactive elements

## Implementation Notes

### State Management

Use local state for wizard flow:

```javascript
const importWizardState = {
  source: null,              // 'ynab' | 'csv' | 'xlsx'
  step: 1,                   // Current step number
  file: null,                // Uploaded file
  rows: [],                  // Parsed rows
  headers: [],               // Column headers
  mapping: {},               // Column mapping
  validationErrors: [],      // Validation errors
  importJobId: null,         // Active import job ID
  progress: 0                // Import progress (0-100)
};
```

### URL Parameters

Track wizard progress in URL:

- `/import?source=ynab&step=2`
- `/import?source=csv&step=3&job=123`

This allows:
- Browser back/forward navigation
- Bookmarking mid-flow (for long imports)
- Sharing import job links
