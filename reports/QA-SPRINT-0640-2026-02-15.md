### üî• BUG-UI-012: Missing "Add" Buttons (CRITICAL) ‚Äî ‚úÖ **FIXED**

**Priority:** P0 CRITICAL  
**Status:** ‚úÖ FIXED (Commit 5f95697)  
**Effort:** 30 minutes

**Problem:**
"Add Asset" button (and likely all "Add" buttons on Assets, Debts, Bills, Income pages) were invisible because JavaScript was not removing the `initially-hidden` CSS class.

**Root Cause:**
```javascript
// app/assets/js/app.js line 3684 (BEFORE FIX)
document.getElementById('pageActions').style.display = currentUser ? '' : 'none';
// Setting display='' doesn't override CSS class .initially-hidden { display: none; }
```

**Fix Applied:**
```javascript
// app/assets/js/app.js line 3684 (AFTER FIX)
const pageActions = document.getElementById('pageActions');
if (currentUser) {
  pageActions.classList.remove('initially-hidden'); // ‚Üê PROPERLY REMOVES CSS CLASS
} else {
  pageActions.style.display = 'none';
}
```

**Impact:**
- **Before:** Users could NOT add assets, investments, debts, bills, or income
- **After:** All "Add" buttons now visible when logged in ‚úÖ

**Git Commit:**
```
5f95697 - fix(critical): BUG-UI-012 - Remove 'initially-hidden' class to show Add buttons
```

### ‚ö†Ô∏è BUG-A11Y-002: Missing Autocomplete Attributes

**Priority:** P3 LOW (Accessibility / Security)  
**Status:** NEW  
**Effort:** 15 minutes

**Problem:**
Password input fields are missing `autocomplete` attributes, preventing browsers from securely auto-filling passwords.

**Console Warning (x4):**
```
[DOM] Input elements should have autocomplete attributes
(suggested: "current-password")
(suggested: "new-password")
```

**Affected Fields:**
- Login modal: email + password inputs
- Sign Up modal: email + password + confirm password inputs
- Reset Password modal: new password + confirm password inputs

**Impact:**
- Password managers may not auto-fill properly (UX)
- WCAG 2.1 Level AA requires autocomplete for authentication (1.3.5)

**Recommended Fix:**
```html
<!-- Login Modal -->
<input type="email" autocomplete="email" ...>
<input type="password" autocomplete="current-password" ...>

<!-- Sign Up Modal -->
<input type="email" autocomplete="email" ...>
<input type="password" autocomplete="new-password" ...>

<!-- Reset Password Modal -->
<input type="password" autocomplete="new-password" ...>
```

**Related:**
- ‚úÖ FC-007: WCAG 2.1 AA accessibility compliance (Done, but missed this)
