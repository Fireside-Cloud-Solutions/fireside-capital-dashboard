## üî¥ Critical Issues

### FC-091: Hardcoded Colors in Notification Categories

**File:** `components.css`  
**Lines:** 275-295  
**Severity:** Medium (maintenance burden)

**Current Code:**
```css
.notification-category.bill {
  background-color: rgba(244, 78, 36, 0.1);
  color: #f44e24;
}

.notification-category.budget {
  background-color: rgba(255, 193, 7, 0.1);
  color: #ffc107;
}

.notification-category.investment {
  background-color: rgba(129, 185, 0, 0.1);
  color: #81b900;
}

.notification-category.system {
  background-color: rgba(1, 164, 239, 0.1);
  color: #01a4ef;
}
```

**Should Be:**
```css
.notification-category.bill {
  background-color: var(--color-primary-light);
  color: var(--color-primary);
}

.notification-category.budget {
  background-color: var(--color-warning-bg);
  color: var(--color-warning);
}

.notification-category.investment {
  background-color: var(--color-accent-light);
  color: var(--color-accent);
}

.notification-category.system {
  background-color: var(--color-secondary-light);
  color: var(--color-secondary);
}
```

**Impact:**  
- Makes theme changes harder (must update multiple files)
- Inconsistent with design token system
- Harder to maintain brand consistency

**Recommendation:** Replace with design tokens.

### FC-092: Toast Container Hardcoded Colors

**File:** `components.css`  
**Lines:** 430-445  
**Severity:** Low (light theme only, may be intentional)

**Current Code:**
```css
.toast-container .toast {
  background-color: #ffffff;
  border-color: #e0e0e0;
}

.toast-container .toast-header {
  background-color: #f5f5f5;
  border-bottom-color: #e0e0e0;
}

.toast-container .toast-body {
  color: #212529;
}

.toast-success .toast-header {
  color: #01a4ef;
}
```

**Analysis:**  
These appear to be light theme overrides for toast notifications. If the app is dark theme only, these can be removed. If light theme support is planned, these should use light theme design tokens.

**Recommendation:**  
- **If dark theme only:** Remove these rules (unused)
- **If light theme planned:** Create light theme tokens in `design-tokens.css`
