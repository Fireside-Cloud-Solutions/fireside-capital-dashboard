## 🐛 Issue Found

### Issue #25: Double Asterisks on Required Fields

**Severity:** Medium (Visual/UX)  
**Pages Affected:** All forms (Assets, Bills, Debts, Income, Investments, etc.)  
**Root Cause:** CSS conflict between HTML `<span class="text-danger">*</span>` and `accessibility.css` line 267 `::after { content: " *"; }`

**Example:**
```html
<!-- HTML has this: -->
<label>Asset Name <span class="text-danger">*</span></label>

<!-- CSS adds this via ::after: -->
.form-label .text-danger::after { content: " *"; }

<!-- Result: "Asset Name * *" (double asterisk) -->
```

**Screenshots:**
- Assets form: Shows "Asset Name * *" and "Asset Type * *"
- Bills form: Shows "Bill Name * *" and "Amount ($) * *"

**Fix Applied:** Commit `75358c3` (Sprint QA 0628)
- Commented out the CSS `::after` rule in `accessibility.css`
- HTML already contains the asterisk, so CSS duplication was unnecessary
- **Status:** Deployed to GitHub, awaiting Azure Static Web App deployment (in progress)
