### FC-090: Transactions empty state not showing on page load ⚠️ CRITICAL
**Status:** FIXED  
**Priority:** High  
**Severity:** Major (UX regression)

**Description:**  
The transactions page was showing an empty table instead of the empty state component when no transactions exist. The `renderTransactionsTable()` function was not being called on page load due to a scope/timing issue with the inline script.

**Root Cause:**  
- The function was defined in `transactions.js` but not accessible from the inline `<script>` tag in `transactions.html`
- The inline DOMContentLoaded handler called `await renderTransactionsTable()` but the function wasn't in the global scope
- This caused a silent failure (no console errors) leaving the page in a partially initialized state

**Impact:**  
- Users see an empty table with headers instead of a helpful empty state message
- No call-to-action to sync transactions from their bank
- Poor first-run experience

**Fix:**  
Moved initialization logic directly into `transactions.js`:
```javascript
// Auto-initialize on page load when transactions.html is present
if (document.getElementById('transactionsTableBody')) {
  document.addEventListener('DOMContentLoaded', async () => {
    try {
      console.log('[Transactions] Initializing page...');
      await renderTransactionsTable();
      console.log('[Transactions] Page initialized successfully');
    } catch (error) {
      console.error('[Transactions] Initialization failed:', error);
    }
  });
}
```

**Testing:**  
- ✅ Empty state now shows correctly with icon, message, and CTA button
- ✅ Manual trigger works (verified via browser console)
