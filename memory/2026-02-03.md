### Bug Fixed

#### ISSUE #4: Filter Button Group Wraps Poorly on Mobile
- **Severity:** üü° MEDIUM (Mobile UX)
- **File:** `bills.html`
- **Location:** Lines 194-204 (filter buttons for "All Bills" / "Subscriptions Only")
- **Issue:** Button group wraps awkwardly on mobile. "Subscriptions Only" text is too long for narrow widths.
- **Fix:** 
  - Changed parent container to responsive flex: `flex-column flex-sm-row`
  - Changed button group to: `d-flex flex-column flex-sm-row gap-2`
  - Buttons now stack vertically on mobile (<576px), horizontal on desktop
  - Added `gap-2` for improved touch target spacing
  - Updated alignment: `align-items-start align-items-sm-center`
- **Commit:** `953130f`
- **Status:** ‚úÖ DEPLOYED

### Implementation Details

**Before:**
```html
<div class="d-flex justify-content-between align-items-center mb-3">
  <h4 class="mb-0"><i class="bi bi-receipt me-2"></i>Recurring Bills</h4>
  <div class="btn-group" role="group" aria-label="Bill filters">
    <!-- buttons -->
  </div>
</div>
```

**After:**
```html
<div class="d-flex flex-column flex-sm-row justify-content-between align-items-start align-items-sm-center gap-3 mb-3">
  <h4 class="mb-0"><i class="bi bi-receipt me-2"></i>Recurring Bills</h4>
  <div class="d-flex flex-column flex-sm-row gap-2" role="group" aria-label="Bill filters">
    <!-- buttons -->
  </div>
</div>
```

### Benefits
- Better mobile UX ‚Äî no awkward wrapping
- Larger touch targets with `gap-2` spacing
- Responsive design follows mobile-first principles
- Maintains horizontal layout on desktop
