### Issue Fixed
**FC-UIUX-003:** Table horizontal overflow on small mobile screens (P1 HIGH)

**Problem:**
- Tables were getting cut off on mobile screens <576px
- No horizontal scroll available
- Data was inaccessible to mobile users
- Affected all pages with tables: Assets, Bills, Debts, Income, Investments, Transactions

**Root Cause:**
- Existing table-responsive handling was at 767.98px breakpoint
- Needed specific fixes for smallest mobile screens (575.98px)
- Tables weren't forcing minimum width to trigger scroll

**Solution Implemented:**
Added comprehensive table scrolling section to `responsive.css` at 575.98px breakpoint:

```css
/* FC-UIUX-003: Table Horizontal Overflow Fix (P1 HIGH) */

/* Ensure table containers allow horizontal scrolling */
.table-card,
.table-responsive {
  overflow-x: auto;
  -webkit-overflow-scrolling: touch;
  width: 100%;
  display: block;
}

/* Force tables to be wide enough to require scrolling */
.table {
  min-width: 650px;
  margin-bottom: 0;
}

/* Ensure table wrapper doesn't clip scroll */
.table-card {
  overflow: visible; /* Allow table-responsive child to scroll */
}

/* Add subtle shadow to indicate scrollable content */
.table-responsive {
  border-radius: var(--radius-md);
  box-shadow: inset 0 -1px 0 var(--color-border-subtle);
}

/* Ensure table cells don't break on small screens */
.table td,
.table th {
  white-space: nowrap;
}

/* Currency and number columns */
.table td[class*="text-end"],
.table td.currency-value {
  white-space: nowrap;
}
```

**Impact:**
- ✅ All 6 pages with tables now have proper horizontal scrolling on mobile
- ✅ Touch-optimized scrolling for iOS/Android (-webkit-overflow-scrolling: touch)
- ✅ Subtle shadow indicates scrollable content
- ✅ No data clipping or truncation
- ✅ Preserves visual design (rounded corners, shadows)
