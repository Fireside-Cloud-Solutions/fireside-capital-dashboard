## Issue Selected: BUG-TX-002

**Title:** Fix Transactions Table Header/Body Column Mismatch  
**Type:** Bug  
**Priority:** P0 (Critical)  
**Effort:** 0.5 hours (5 minutes)  
**Reason:** Only actionable P0 work under 1 hour (per AGENTS.md delegation rules)

### Problem
Transactions table header defines 5 columns, but the 5th column header says "Confidence" when it actually displays the "Pending" badge. Confidence score is embedded under the Category dropdown, not in its own column.

**Current Header:**
```
Date | Description | Category | Amount | Confidence
```

**Actual tbody columns:**
```
Date | Merchant | Category (+ confidence) | Amount | Pending badge
```

**Impact:**
- Screen readers announce incorrect column names
- Visual misalignment
- Semantic HTML violation
- WCAG accessibility issue

### Fix Applied
**File:** `app/transactions.html` (line 212)  
**Change:** `<th>Confidence</th>` ‚Üí `<th>Status</th>`

**Rationale:** The 5th column displays transaction status (Pending badge), not a confidence score. Confidence is shown as a small text note under the category dropdown, not in a dedicated column.

**Commit:** `9f37f69` - `fix(transactions): BUG-TX-002 - Change 'Confidence' header to 'Status' for semantic accuracy`
