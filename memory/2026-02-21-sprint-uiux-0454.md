## Global Issue Identified

**BUG-UIUX-MODAL-FORM-SPACING-001** (P2, 2 hours)

**Problem:**
All modal forms across the app use Bootstrap's default `mb-3` (16px) for ALL spacing. According to Gestalt proximity principle, labels should be closer to their inputs (4px) while fields should have larger gaps (16px).

**Current State:**
```html
<div class="mb-3">
  <label class="form-label">Email address</label>
  <input type="email" class="form-control">
</div>
```
- Label to input: 16px (too much)
- Input to next field: 16px (correct)

**Correct State:**
```html
<div class="mb-3">
  <label class="form-label mb-1">Email address</label>
  <input type="email" class="form-control">
</div>
```

**Affected Files:** 10+ HTML pages (all modals)  
**Effort:** 2 hours (global find/replace + testing)  
**Impact:** Better visual grouping, improved form scanning

## Design System Analysis

### Strengths (A grade)

1. **Color System:** Excellent adherence to tri-color hierarchy
   - Flame Orange (PRIMARY) used correctly for main CTAs
   - Sky Blue (SECONDARY) for medium-impact actions
   - Lime Green (SUCCESS) for positive states
   - Proper use of design tokens from `design-tokens.css`

2. **Typography:** Strong implementation
   - Source Serif 4 for headings (correct)
   - Inter for body text (correct)
   - Mostly uses rem units (WCAG 1.4.4 compliant)
   - Good visual hierarchy (32px → 24px → 16px)

3. **Accessibility:** Solid foundation
   - Skip links on all pages
   - Proper ARIA labels
   - Good heading hierarchy
   - 44px minimum touch targets

4. **Spacing Grid:** Mostly consistent
   - 8px base grid followed in most areas
   - Proper use of mb-3, mb-4, gap-2, gap-3

### Gaps (B- grade)

1. **Empty States:** Not standardized
   - Some pages have excellent empty states (Assets, Debts, Income)
   - Others missing or inconsistent (Friends search, Bills initially)
   - Need to create reusable component pattern

2. **Button Active States:** Inconsistent
   - Operations toggle uses Bootstrap default (poor contrast)
   - Other pages use custom states
   - Need global active state system

3. **Form Spacing:** Global inconsistency
   - All modals use mb-3 for label-to-input
   - Should be mb-1 for better visual grouping
   - Large refactor needed (10+ files)

4. **Ad-hoc Spacing:** Some violations
   - Transactions Quick Ranges uses 12px margin
   - Should use 8px grid (mb-8, mb-16, mb-24)

## Key Learnings

1. **Empty States Are Critical**
   - Friends page demonstrates the importance of empty states
   - Without placeholder, users are confused about functionality
   - Always provide initial state + loading state + no-results state

2. **Gestalt Proximity Principle Matters**
   - Labels should be close to their inputs (mb-1 = 4px)
   - Fields should be spaced apart (mb-3 = 16px)
   - This visual grouping helps users scan forms faster

3. **Active States Need High Contrast**
   - Bootstrap defaults don't have enough contrast in dark mode
   - Custom active states with brand colors (--color-secondary) + shadow work better
   - Important for toggle buttons where state is primary information

4. **8px Grid System Benefits**
   - Pages that follow the grid (Assets, Bills) feel more polished
   - Ad-hoc spacing values (12px, 10px) create visual inconsistency
   - Design tokens make this easy to enforce

5. **Code Documentation Reduces Confusion**
   - Hidden-by-default elements (pagination) need comments
   - Future developers (or AI agents!) benefit from context
   - Semantic attributes (data-state) also help
