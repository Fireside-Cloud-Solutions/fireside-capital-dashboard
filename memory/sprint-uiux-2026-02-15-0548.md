## New Issues Found

### ❌ BUG-DEBTS-001: Button Hierarchy Violation — P1 HIGH

**Location:** debts.html line 106  
**Problem:** "Add Debt" button on main page uses `btn-primary`, violating design philosophy

**Current Code:**
```html
<button class="btn btn-primary" data-bs-toggle="modal" data-bs-target="#addDebtModal">
  <i class="bi bi-plus-circle"></i> Add Debt
</button>
```

**Fix:**
```html
<button class="btn btn-secondary" data-bs-toggle="modal" data-bs-target="#addDebtModal">
  <i class="bi bi-plus-circle"></i> Add Debt
</button>
```

**Rationale:**
- Design philosophy: "PRIMARY actions - 1 per page max"
- Modals can use btn-primary, but main page actions should use btn-secondary
- Assets page correctly uses btn-secondary for "Add Asset" button
- Inconsistency creates visual hierarchy confusion

**Impact:** Visual hierarchy broken — primary orange should be reserved for modal forward actions

**Effort:** < 5 minutes (one-line change)
