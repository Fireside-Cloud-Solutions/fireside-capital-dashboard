## What I Learned

### Dashboard Audit Revealed Missed Patterns
- **Observation:** Session 0446 marked all 11/11 pages as 100% audited
- **Reality:** Dashboard audit (Session 0525) found 7 NEW issues
- **Conclusion:** Initial audits focused on functional bugs, missed design system and performance patterns
- **Action:** Recommend re-audit for button hierarchy, skeleton animations, script bundling across all 11 pages

### Design Tokens vs. Actual Usage
- **Finding:** `--weight-regular: 400` defined in design-tokens.css but NEVER used
- **Implication:** Design tokens can document "available" options without being actively used
- **Lesson:** Always verify token usage before assuming it's needed
- **Applied:** Safely removed Inter:400 from font import

### Static Modals Need Explicit Exit Routes
- **Best Practice:** If `data-bs-backdrop="static"`, always provide Cancel/Close button
- **Rationale:** User could be trapped if error occurs (network failure, invalid input)
- **Pattern:** All modals should have either dismissible backdrop OR explicit Cancel button (never neither)
