## Bug Fixed

### BUG-UIUX-MODAL-FORM-SPACING-001 (P2, 10 min)

**Issue:**  
All modal form fields used Bootstrap default `mb-3` (16px) spacing between labels and inputs. This violated the Gestalt proximity principle â€” labels should be visually closer to their inputs than to previous fields.

**Fix:**  
Added `mb-1` (4px) class to all 186 form labels across 12 HTML pages using PowerShell batch find/replace:

```powershell
cd C:\Users\chuba\fireside-capital\app
Get-ChildItem -Filter *.html | ForEach-Object { 
  (Get-Content $_.FullName) -replace 'class="form-label"', 'class="form-label mb-1"' | Set-Content $_.FullName 
}
```

**Before:** `<label for="billName" class="form-label">Bill Name</label>`  
**After:** `<label for="billName" class="form-label mb-1">Bill Name</label>`

**Result:**  
- Better visual hierarchy: labels now clearly associated with their inputs
- Follows 8px grid system: 4px micro-spacing for related elements
- Improved form readability across all modal forms (Add Asset, Add Bill, Login, Sign Up, etc.)
