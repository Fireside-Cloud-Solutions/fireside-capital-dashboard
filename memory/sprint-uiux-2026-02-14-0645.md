### Critical Issue (P1)

**FC-131: No Pagination or Virtual Scrolling**
- **Problem:** Table will render ALL transactions at once. With 500+ transactions, this will cause severe performance degradation (slow loads, janky scrolling, memory bloat, mobile unusable).
- **Fix:** Implement server-side pagination with page controls (Previous/1-2-3/Next), items per page selector (10/25/50/100), showing range indicator ("Showing 1-25 of 347 transactions")
- **Estimated Effort:** 4-5 hours
- **Impact:** BLOCKING for scale — app unusable with 1 year of transactions

### High-Priority Issues (P2)

1. **FC-128: Button Hierarchy Violation**
   - "Sync from Bank" uses `btn-secondary` instead of `btn-primary`
   - Violates established pattern (FC-043)
   - Fix: Change to `btn-primary` (core action)
   - Effort: 15 minutes

2. **FC-129: Missing Table Skeleton Loaders**
   - Generic spinner instead of skeleton rows
   - Inconsistent with Dashboard loading pattern (FC-056)
   - Fix: Create `.skeleton-table-row` component with 5 shimmer rows
   - Effort: 2 hours

3. **FC-130: Status Column Not Implemented**
   - Table header declares "Status" column but no data renders
   - Fix: Implement pending/cleared/failed badges using Plaid `pending` field
   - Effort: 2 hours

4. **FC-134: Mobile Filter Layout Not Optimized**
   - Three-column filter layout may stack poorly on mobile
   - Fix: Test and add `col-12` for mobile breakpoint
   - Effort: 1 hour

5. **FC-135: Empty State CTA Verification**
   - Need to verify "Sync from Bank" buttons trigger same action
   - Fix: Code review + testing
   - Effort: 30 minutes

### Low-Priority Issues (P3)

1. **FC-132: No Active Filter Indicator**
   - Users can't tell if filters are applied
   - Fix: Add badge count ("3 filters active") + blue borders
   - Effort: 1 hour

2. **FC-133: Auto-Categorize Button Hierarchy Unclear**
   - Uses `btn-outline-secondary` but importance unclear
   - Fix: Document design decision (secondary vs tertiary action)
   - Effort: 30 minutes
