## 🎯 How `renderUpcomingPayments()` Works

**Location:** `app/assets/js/app.js` lines 3895-3920

**Logic:**
1. Get all bills + debts + shared bills
2. Filter for items due within next 7 days
3. Sort by due date
4. **Empty state check:**
   ```javascript
   if (upcoming.length === 0) {
     if (typeof generateEmptyStateHTML === 'function') {
       c.innerHTML = generateEmptyStateHTML('upcomingPayments'); // NOW WORKS ✅
     } else {
       // Fallback (was used before fix)
       c.innerHTML = '<p class="text-muted fst-italic">No upcoming payments this week.</p>';
     }
     return;
   }
   ```
5. Render upcoming payment list

**Why It Was Broken:**
- `generateEmptyStateHTML()` is defined in `empty-states.js`
- `empty-states.js` was NOT loaded on Dashboard (`index.html`)
- Function check failed → fallback text rendered

**Why It's Fixed:**
- `empty-states.js` now loaded BEFORE `app.js` executes
- `generateEmptyStateHTML()` available when `renderUpcomingPayments()` runs
- Styled empty state component renders correctly
