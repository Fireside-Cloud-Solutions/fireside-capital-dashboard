## Task Completed

**BUG-UI-TYPE-SYSTEMIC-H1-001** — Fixed systemic WCAG 2.4.6 violation

### Problem
All 11 pages (except Settings) were using `<h2>` instead of `<h1>` for their main page titles, violating WCAG 2.1 AA Success Criterion 2.4.6. Screen readers expect exactly ONE h1 tag per page to identify the main content purpose.

### Solution
1. **Updated CSS** — Added h1 support to existing `.page-header h2` selectors in main.css:
   - Line 204: `.page-header h1, .page-header h2` (desktop)
   - Line 251: `.page-header h1, .page-header h2` (responsive)

2. **Updated HTML** — Changed 11 page titles from h2 to h1:
   - assets.html
   - bills.html
   - budget.html
   - index.html (dashboard)
   - debts.html
   - friends.html
   - income.html
   - investments.html
   - operations.html (preserved icon in heading)
   - reports.html
   - transactions.html
   - settings.html (already fixed in commit ac37738)
