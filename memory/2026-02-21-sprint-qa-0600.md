### 3. CRITICAL BUG FOUND: BUG-SYSTEMIC-HIDDEN-ACTIONS-001

#### Summary
**Title:** Page action buttons hidden on load across 9 pages  
**Priority:** P1 (High)  
**Effort:** XS (15 min batch fix)  
**Impact:** 75% of pages affected

#### Affected Pages (9/12)
1. assets.html (line 91)
2. bills.html (line 90)
3. budget.html (line 90)
4. debts.html (line 90)
5. friends.html (line 90)
6. income.html (line 90)
7. investments.html (line 90)
8. reports.html (line 97)
9. transactions.html (line 90)

**NOT Affected:**
- Dashboard (no pageActions div)
- Operations (different toolbar structure)
- Settings (no pageActions div)

#### Problem
`<div id="pageActions" class="initially-hidden">` causes primary action buttons to:
- ‚ùå Hide on page load
- ‚ùå Appear with 200-500ms delay (FOUC)
- ‚ùå Poor UX - users can't access actions immediately

#### Root Cause
Double-hiding mechanism:
1. Buttons wrapped in `#loggedInState` (auth gate) ‚úÖ Needed
2. Buttons also wrapped in `.initially-hidden` ‚ùå Unnecessary

#### Fix
Remove `class="initially-hidden"` from `<div id="pageActions">` in 9 files.

Buttons are already protected by `#loggedInState` wrapper.

**PowerShell batch fix:**
```powershell
$files = @('assets.html', 'bills.html', 'budget.html', 'debts.html', 'friends.html', 'income.html', 'investments.html', 'reports.html', 'transactions.html')
$files | ForEach-Object {
  $path = "C:\Users\chuba\fireside-capital\app\$_"
  (Get-Content $path) -replace '<div id="pageActions" class="initially-hidden">', '<div id="pageActions">' | Set-Content $path
}
```
