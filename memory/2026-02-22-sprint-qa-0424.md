## Lessons Learned

### QA Process Improvement Needed
- **Problem:** Sprint QA 0400 falsely verified bug as fixed
- **Root Cause:** Tested wrong thing (Reports page receiving data) instead of verifying migration executed
- **Fix:** Always verify database changes actually applied, not just that code expects them

### Verification Checklist for Database Bugs
1. ✅ Verify migration file exists
2. ✅ Execute migration on database
3. ✅ Query database to confirm schema changes
4. ✅ Test affected pages with console open
5. ✅ Confirm NO error messages related to bug
6. ✅ Verify new functionality working (snapshots saving)
